1. 🏠 FLUJO PRINCIPAL: Acceso y Navegación

🔑 Arquitectura de Datos

Todas las APIs se consultan en paralelo en cada solicitud. En la home se muestra el promedio calculado de todas las fuentes disponibles. Las fuentes que fallen se excluyen del cálculo sin afectar el funcionamiento.

📍 Puntos de Entrada

  • Home principal de la aplicación → Footer "Clima"

👤 Actores

  • Usuario ganadero con establecimientos cargados
  • Usuario ganadero sin establecimientos
  • Usuario nuevo

🛤️ Flujo Principal (Happy Path)

flowchart TD A([Usuario en Home Principal]) --> B[Click en Footer: Clima] B --> C{¿Tiene establecimientos?} C -->|Sí| D[Mostrar Home Clima] D --> E[[Consultar 5 APIs en paralelo]] E --> F[Mostrar Establecimiento Principal] F --> G[Datos PROMEDIO: temp, condición, viento, lluvia] G --> H[Mostrar timestamp actualización] H --> I[Swipe horizontal entre establecimientos] I --> J[Tap en establecimiento específico] J --> K[Ver Detalle: TODAS las fuentes + promedio] K --> L([Fin: Usuario ve tabla comparación]) C -->|No| M[Mostrar mensaje sin establecimientos] M --> N[CTA: Agregar Establecimiento] N --> O[Formulario establecimiento] O --> P{¿Datos válidos?} P -->|Sí| Q[[Crear establecimiento]] Q --> R[[📱 MSG-01: Establecimiento creado]] R --> D P -->|No| S[Mostrar errores validación] S --> O

❌ Estados de Error

  • Algunas APIs fallan: Se calculan promedios con las APIs disponibles
  • Todas las APIs fallan: Mostrar mensaje de error + datos cache si disponible
  • Establecimiento sin coordenadas: Redirigir a configuración ubicación

💫 Interacciones Clave

  • Swipe horizontal: Navegar entre 2+ establecimientos (cada uno con promedio calculado)
  • Tap establecimiento: Acceso a detalle con tabla de todas las fuentes
  • Pull to refresh: Reconsultar las 5 APIs y recalcular promedios
  • Timestamp: Indicador visual última actualización de todas las fuentes

2. 📊 FLUJO: Vista Detalle con Todas las Fuentes

📋 Tabla de Comparación (como en la imagen)

La vista de detalle muestra por defecto una tabla con todas las fuentes simultáneamente, organizada por franjas horarias. No es un botón separado, es la vista principal del establecimiento.

📍 Punto de Entrada

  • Detalle de establecimiento → Vista por defecto (tabla de comparación siempre visible)

🛤️ Flujo Principal

flowchart TD A([Usuario en Detalle Establecimiento]) --> B[Vista por defecto: Tabla comparación] B --> C[[Consultar 5 APIs paralelas SIEMPRE]] C --> D[Tabla con filas por fuente] D --> E[YR | Tomorrow | Google Weather | AccuWeather | Weather Channel] E --> F[Columnas: mm lluvia | temp | humedad | viento | aplicación] F --> G[Fila "Promedio" calculada automáticamente] G --> H[Fuentes caídas: en gris o ausentes] H --> I[Navegación DUAL disponible] I --> J[Usuario cambia DÍA: ←→ Hoy, Mañana, Pasado...📏] J --> K[Usuario cambia FRANJA: Mañana | Tarde | Noche 🔍] K --> L[Indicador: "Jueves - Tarde" visible] L --> M[Solo se muestra UNA tabla por día+franja] M --> N([Fin: Vista compacta y navegable])

📊 Datos Mostrados por Fuente (como en la imagen)

  • Precipitaciones: cantidad en mm
  • Temperatura: Mín/Máx (ej: 22°/22°)
  • Humedad: Porcentaje (ej: 47%)
  • Viento: Velocidad km/h + dirección (ej: 27 km/h ↕️)
  • Condición de aplicación: 🔴 No aplicar / 🟡 Cautela / 🟢 Aplicar

🗂️ Estructura de la Tabla

  • Filas: Una por cada fuente (YR, Tomorrow, Google Weather, AccuWeather, etc.) + fila "Promedio"
  • Columnas: mm lluvia | temp | humedad % | viento | aplicación
  • Vista por día: Solo un día a la vez (hoy + 7 días disponibles)
  • Vista por franja: Solo una franja a la vez (Mañana 6-12, Tarde 12-18, Noche 18-6)
  • Navegación dual: Controles separados para día y franja
  • Indicador activo: "Hoy - Mañana", "Viernes - Noche", etc.

4. 🔔 FLUJO: Configuración de Notificaciones

📍 Puntos de Entrada

  • Home clima → Botón "Configurar Notificaciones"
  • Detalle establecimiento → Settings → Notificaciones

🛤️ Flujo Principal

flowchart TD A([Usuario accede a configuración]) --> B[Pantalla Configuración Notificaciones] B --> C[Toggle general: Notificaciones ON/OFF] C --> D{¿Notificaciones activadas?} D -->|Sí| E[Configurar tipos específicos] E --> F[Toggle Lluvia + Umbral mm] F --> G[Slider umbral: 0-50mm] G --> H[Toggle Viento + Umbral km/h] H --> I[Slider umbral: 0-100 km/h] I --> J[Toggle Condición Aplicación] J --> K[Selector establecimientos: Todos/Específicos] K --> L[Nota: Umbrales evalúan PROMEDIO de fuentes] L --> M[Botón Guardar Configuración] M --> N[[Validar configuración backend]] N --> O[[📱 MSG-02: Configuración guardada]] O --> P([Fin: Notificaciones configuradas]) D -->|No| Q[Deshabilitar todas las notificaciones] Q --> M

🔧 Tipos de Notificaciones Configurables

  • Lluvia: Umbral en mm esperados (0-50mm) - evaluado sobre promedio
  • Viento: Umbral en km/h (0-100 km/h) - evaluado sobre promedio
  • Condición de Aplicación: Cambio a 🔴 No Aplicar - basado en promedio

⚙️ Configuraciones Avanzadas

  • Horarios: 6:00-22:00 (evitar notificaciones nocturnas)
  • Frecuencia: Máximo 1 notificación por evento por día
  • Alcance: Todos los establecimientos o selección específica

📍 Punto de Entrada

  • Home clima → Botón "Buscar por Ubicación"

🛤️ Flujo Principal

flowchart TD A([Usuario en Home Clima]) --> B[Tap "Buscar por Ubicación"] B --> C[Selector Provincia] C --> D[Lista completa provincias argentinas] D --> E[Usuario selecciona provincia] E --> F[[Cargar partidos de la provincia]] F --> G[Lista partidos filtrada] G --> H[Usuario selecciona partido] H --> I[[Obtener coordenadas del partido]] I --> J{¿Coordenadas encontradas?} J -->|Sí| K[[Consultar 5 APIs para ubicación]] K --> L[Mostrar clima PROMEDIO para ubicación] L --> M[Opción: Guardar como establecimiento temporal] M --> N{¿Usuario guarda?} N -->|Sí| O[Formulario establecimiento básico] O --> P[[Crear establecimiento temporal]] P --> Q[[📱 MSG-03: Ubicación guardada]] Q --> R([Fin: Nueva ubicación disponible]) N -->|No| R J -->|No| S[Error: Ubicación no encontrada] S --> T[CTA: Reintentar o elegir otra] T --> G

📊 Datos Requeridos

  • Base de datos: 23 provincias + partidos/departamentos
  • Coordenadas: Lat/Lng por centro geográfico del partido
  • Validación: Existencia de partido en provincia seleccionada

📍 Establecimiento Temporal

  • Nombre: "[Provincia] - [Partido]"
  • Tipo: "Ubicación consultada"
  • Persistencia: Se mantiene en la lista de establecimientos

6. 🗺️ FLUJO: Gestión de Coordenadas

📍 Puntos de Entrada

  • Establecimiento sin coordenadas → Auto-redirect
  • Detalle establecimiento → "Editar Ubicación"

🛤️ Flujo Principal

flowchart TD A([Establecimiento sin coordenadas]) --> B[Pantalla: Ubicación no configurada] B --> C[Mensaje: Necesaria para pronóstico preciso] C --> D[Opciones de configuración] D --> E{¿Método seleccionado?} E -->|Seleccionar por mapa| F[Abrir mapa interactivo] F --> G[Usuario toca ubicación en mapa] G --> H[Confirmar coordenadas seleccionadas] E -->|Ingresar manualmente| I[Campos Latitud/Longitud] I --> J[Validación formato coordenadas] J --> K{¿Formato correcto?} K -->|Sí| H K -->|No| L[Error: Formato inválido] L --> I H --> M[[Validar coordenadas con API]] M --> N{¿Coordenadas válidas?} N -->|Sí| O[[Guardar coordenadas establecimiento]] O --> P[[📱 MSG-04: Ubicación configurada]] P --> Q[[Consultar 5 APIs para nueva ubicación]] Q --> R[Mostrar clima promedio para nueva ubicación] R --> S([Fin: Coordenadas guardadas]) N -->|No| T[Error: Coordenadas fuera de Argentina] T --> U[CTA: Corregir ubicación] U --> F

✅ Validaciones de Coordenadas

  • Rango Argentina: Lat -55 a -22, Lng -74 a -53
  • Precisión: Hasta 6 decimales
  • Formato: Decimal degrees (no DMS)

🔧 Fuentes de Coordenadas

  • Mapa interactivo: Google Maps integrado
  • Ingreso manual: Campos numéricos con validación
  • Sugerencias: Basadas en partido/provincia conocidos

📍 Punto de Entrada

  • Home clima → Navegación establecimientos → "+" Agregar

🛤️ Flujo Principal

flowchart TD A([Usuario en navegación establecimientos]) --> B[Tap botón "+" Agregar] B --> C[Formulario Nuevo Establecimiento] C --> D[Campo: Nombre establecimiento*] D --> E[Selector: Provincia*] E --> F[Selector: Partido* filtrado] F --> G[Selector: Sociedad asociada*] G --> H[Campo opcional: Observaciones] H --> I[Botón: Crear Establecimiento] I --> J[[Validar datos requeridos]] J --> K{¿Todos los campos OK?} K -->|Sí| L[[Crear establecimiento en BD]] L --> M[[Obtener coordenadas por partido]] M --> N[[📱 MSG-05: Establecimiento creado]] N --> O[Establecimiento disponible inmediatamente] O --> P[[Consultar 5 APIs para nuevo establecimiento]] P --> Q[Mostrar clima promedio del nuevo establecimiento] Q --> R([Fin: Establecimiento operativo]) K -->|No| S[Mostrar errores específicos] S --> T{¿Error nombre duplicado?} T -->|Sí| U[Error: Nombre ya existe en sociedad] T -->|No| V[Errores de campos requeridos] U --> D V --> C

📝 Campos del Formulario

  • Nombre: Texto libre, único por sociedad
  • Provincia: Selector con 23 provincias
  • Partido: Filtrado dinámico por provincia
  • Sociedad: Lista de sociedades del usuario
  • Observaciones: Campo opcional, 200 caracteres máx

✅ Validaciones

  • Nombre único: Por sociedad asociada
  • Campos requeridos: Nombre, provincia, partido, sociedad
  • Integración: Con sistema existente gestión establecimientos

8. ⚠️ FLUJOS de Error y Contingencia

⚡ Nueva Estrategia: No hay APIs backup

Todas las APIs se consultan en paralelo. Si algunas fallan, se calcula el promedio con las que respondan. Solo si todas fallan se usa cache o se muestra error.

🔌 8.1 Error de APIs Externas

flowchart TD A([Solicitud datos climáticos]) --> B[[Llamada paralela a 5 APIs]] B --> C[AccuWeather | YR | Weather Channel | Google | Tomorrow] C --> D{¿Al menos 1 API responde?} D -->|Sí| E[Calcular promedio con APIs disponibles] E --> F[Mostrar fuentes caídas en gris] F --> G[Continuar funcionamiento normal] D -->|No| H[Reintento paralelo x3 a todas] H --> I{¿Alguna API responde?} I -->|Sí| E I -->|No| J[Modo degradado: datos cache] J --> K{¿Cache disponible?} K -->|Sí| L[Mostrar datos desactualizados] K -->|No| M[Error: Sin datos disponibles] M --> N[CTA: Reintentar más tarde]

📶 8.2 Error de Conectividad

flowchart TD A([Usuario sin conexión]) --> B{¿Cache local disponible?} B -->|Sí| C[Mostrar datos cache con promedios] C --> D[Banner: Datos no actualizados] D --> E[Auto-refresh: consultar 5 APIs cuando vuelva conexión] B -->|No| F[Pantalla: Sin conexión] F --> G[Mensaje explicativo] G --> H[CTA: Verificar conexión] H --> I[CTA: Reintentar]

📍 8.3 Error de Geolocalización

flowchart TD A([Establecimiento sin coordenadas]) --> B[Intentar geolocalización por partido] B --> C{¿Coordenadas encontradas?} C -->|No| D[Mostrar configuración manual] D --> E[Usuario configura ubicación] E --> F{¿Configuración exitosa?} F -->|Sí| G[[Consultar 5 APIs para nueva ubicación]] G --> H[Continuar flujo normal con promedio] F -->|No| I[Mantener sin datos climáticos] I --> J[Mensaje: Configurar para ver clima]
